# Privacy and Safety

ShmagStick runs locally and does not upload scan data. Scanning works offline.

## Collected diagnostic data

- Hardware models and capacities.
- Operating-system version and update state.
- Aggregate CPU, memory, disk, battery, network, and thermal values.
- Process names and aggregate resource use for top consumers.
- Startup/login item names and selected service/task names.
- Counts and short summaries of relevant crash/hardware events.
- Browser extension counts and cache-size estimates.
- Native security/firewall state and suspicious startup-item basenames.

## Data never intentionally collected

- Browsing history, visited URLs, or browser content.
- Passwords, cookies, tokens, keys, or credentials.
- Documents, photos, message content, or personal-file contents.
- Downloads/Desktop inventories.
- Network packet contents.

Reports and logs remain beside the application when the media is writable. Logs rotate at approximately 1 MB and retain three backups. Exception logs use operation/type details and avoid personal-file paths.

Report filenames include a sanitized computer hostname and timestamp so technicians can distinguish reports. The same machine identifier may appear in the report's machine summary.

ShmagStick performs no cleanup, deletion, repair, service modification, registry modification, software installation, update installation, malware removal, or destructive benchmark. Shopping links open only when clicked.

Generated Amazon upgrade links include the configured Amazon Associates tag `shmagstick-20`. Amazon determines whether a click qualifies for commission.
